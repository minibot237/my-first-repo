# Progress: March 8, 2026

## What We Built

Starting from an empty repo with just a planning doc, we built a working container-based agent system with a real-time web dashboard.

### Container Hello World (Step 1 + Step 2 complete)

**Two container runtimes** behind a shared `ContainerRuntime` interface:

- **Apple Containers** (`brew install container`) — uses `--publish-socket` for vsock-bridged IPC. Agent listens inside the VM, supervisor connects from host. Required macOS 26, Xcode, Rosetta 2, and `container system start`.
- **Docker** — uses `-v` bind-mount for unix socket IPC, `--network none` for isolation. Supervisor listens on host, agent connects inside container. Docker socket is never mounted.

**Protocol:** Length-prefixed JSON (4-byte big-endian uint32 + UTF-8 JSON body) over unix sockets. `MessageReader` class handles stream reassembly.

**Agent modes:** `AGENT_MODE=listen` (Apple Containers default) or `AGENT_MODE=connect` (Docker). Same agent binary, different socket direction.

### Two-Channel Model

Each VM has two logical channels over the single socket, separated by a `channel` field on every message:

- **Ops channel** — heartbeats (every 3s), nudge/nudge_ack, status, control
- **Work channel** — hello/hello_ack, web_fetch/web_fetch_response

### Real-Time Web Dashboard

Single-page web UI served by the supervisor on port 9100:

- **Left column (30%):** Command panel with input box. Commands: `nudge [id|*]`, `filter [text]`, `status`, `clear`
- **Right column (70%):** VM panels with ops and work channels **side by side**
- **Focus mode:** Click a VM header to expand to full height with detailed payloads
- **WebSocket:** `ws` library, broadcasts `DashboardEvent` objects from an internal `EventBus` (`EventEmitter`)
- **Auto-reconnect** on disconnect, buffers last 200 events for late-joining clients
- Dark theme, monospace, color-coded (green=ops success, blue=work, yellow=control, red=errors)

### Dumb Agent Demo

The containerized agent:
- Sends `hello` on work channel at startup, receives `hello_ack`
- Sends heartbeat on ops channel every 3 seconds
- Responds to `nudge` command with a `nudge_ack` (ops) + fake `web_fetch` request (work)
- Supervisor responds with a canned `web_fetch_response`

## Key Technical Notes

- **Apple Containers connection timing:** The `--publish-socket` bridge creates the host socket file before the agent calls `listen()` inside the VM. The supervisor retries connection (up to 20 attempts, 500ms apart) and resolves only on first data received, not on TCP connect.
- **Build workflow:** `npx tsc` → copy `dist/shared` + `dist/container` into `container-image/` → copy `index.html` to `dist/host/dashboard-ui/` → supervisor builds image automatically via runtime.
- **HTML not compiled by tsc:** The dashboard HTML must be manually copied to `dist/host/dashboard-ui/` after changes.
- **Dependencies:** `typescript`, `@types/node`, `ws`, `@types/ws`
- **Runtime selection:** `RUNTIME=apple-containers|docker` env var

## Future Architecture (Documented, Not Built)

The supervisor will become a **message router and security gateway**:

- Multiple agent VMs, each with ops + work channels
- **Safety VM** — honeypot sandbox with fake high-value targets (bitcoin keys, passwords.txt, fake network stack). Runs a small local model (4B/7B) to evaluate content chunks for prompt injection. Captures and freezes on trigger.
- **Content pipeline:** raw HTML → strip/markdown → regex scan → chunk with overlap → Safety VM
- **LLM proxy:** routes agent requests to LM Studio local models or external APIs
- **Filtered internet proxy:** supervisor mediates all external access
- **Timer-driven tasks:** email checking, periodic scans
