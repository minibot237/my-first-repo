## Trust Store Seeding — IN PROGRESS

Pre-classifier now computes recommended initial fit values for new sources. Three pieces built:

1. **`computeInitialFit(tier, authScore)`** in `pre-classify.ts` — maps tier + auth to seed value:
   - skip (auth=0): 0.1 — untrusted garbage
   - trusted (auth=1 + ESP): 0.4 — authenticated marketing
   - full, auth>0: 0.3 — some auth, unknown sender type
   - full, auth=0: 0.2 — no auth, unknown sender
   - Never above 0.5 — you earn trust, you don't start with it.

2. **`seedIfNew()`** on TrustStore (`store.ts`) — creates entry only for new sources, no-op if source exists. Logs as `seed: <reason>`, emits dashboard event.

3. **`initialFit` + `initialFitReason`** on `PipelineResult` — pipeline computes and returns, supervisor applies.

**Not yet wired:** Supervisor doesn't consume `evaluatePipeline()` yet — it uses `evaluateContent()` directly for the `canary_evaluate` dashboard command. The seeding will activate when the supervisor starts running the full pipeline on ingested content. The smoke test already exercises the pipeline and will show `initialFit` values.

## Canary Autoresearch — IN PROGRESS

Autonomous prompt optimization for the canary classifier, inspired by Karpathy's [autoresearch](https://github.com/karpathy/autoresearch). The core idea: fixed evaluation budget + automatic keep/revert + autonomous researcher agent = overnight prompt tuning on the Studio.

### Infrastructure Built
- **Configurable prompts directory** — `CANARY_CONFIG.promptsDir` swaps prompt files without touching prod. Cache auto-invalidates on dir change.
- **Adaptive observation scoring** — `computeObservationScore()` checks which fields the response format actually asks for, so minimal response formats don't get penalized for missing fields the format never requested.
- **A/B comparison runner** — `src/host/canary/experiment-ab.ts` runs full smoke suite with prod (A) and experiment (B) prompts, outputs comparison table with disagreements and token savings.

### Experiment 1: `prompts-minimal` — Too aggressive
Gutted system prompt to 2 sentences, response format to `{safe, confidence}` only.
- **Token savings confirmed**: output -72.6%, input -14.9%, 21% faster, 100% obs scores.
- **Accuracy regression**: 34 disagreements / 86 emails. Model lost nuance about marketing/urgency being normal. Nearly all were safe→flagged. The explicit safe-list in the prod prompt is doing critical work.

### Experiment 2: `prompts-lean` — Worse than minimal
Kept the safe-list but restructured into SAFE/UNSAFE bullet lists. 47 disagreements (all safe→flagged) — worse than both prod and minimal. The structured list format made the model more trigger-happy, not less. The prod prompt's prose framing works better than structured lists for this 3B model.

### Key Findings
- **Response format cut is free**: 72%+ output token reduction, zero accuracy impact. `{safe, confidence}` is enough.
- **System prompt structure is delicate**: same content, different format = wildly different accuracy. This is exactly the search space an autonomous researcher should explore — hundreds of variations overnight.
- **The prod prompt's prose framing is doing real work**: "Normal human-to-human content is SAFE, even if..." is better than SAFE/UNSAFE lists.

### Experiment 3: `prompts-hybrid` — SHIP IT
Prod system prompt verbatim + 2-field response format. The control experiment.
- **Output tokens**: 5228 → 1376 (73.7% reduction)
- **7 disagreements / 86 emails** — split 3/4 (noise, not signal). Compare: minimal had 34, lean had 47.
- **Confirms**: response format cut is safe to ship. System prompt is untouched.

Task doc: `.local/tasks/canary-autoresearch.md`
